UNIT I - Database Systems and Design

Introduction to Database and System Architecture

Database Systems and their Applications

Database systems manage data efficiently and allow users to perform multiple tasks with ease. Applications include banking, airlines, universities, sales, and telecommunication.

Database Vs File System

  • File System: Manual management of data, no standardized querying.
  • Database System: Centralized management, controlled redundancy, security, and concurrency control.

View of Data

  • Physical Level: How data is stored.
  • Logical Level: What data is stored and relationships among the data.
  • View Level: User interaction with the data.

Data Models

  • Relational Model
  • Entity-Relationship Model
  • Object-Based Data Model
  • Semi-Structured Data Model

Database Languages

  • DDL (Data Definition Language): Defines the database structure (e.g., CREATE, DROP, ALTER).
  • DML (Data Manipulation Language): Manipulates data (e.g., SELECT, INSERT, UPDATE, DELETE).

Transaction Management

A transaction is a unit of a program that accesses and possibly modifies data. Ensures properties: Atomicity, Consistency, Isolation, Durability (ACID properties).

Database Users and Administrators

  • Users: Application programmers, sophisticated users, casual users, and naive users.
  • DBA (Database Administrator): Responsible for authorization, backup, performance tuning, and database design.

Database System Structure

The architecture typically includes Query Processor, Storage Manager, and Transaction Manager components.

Introduction to Database Design

ER Diagrams

Entity-Relationship Diagrams graphically represent entities, relationships, and attributes.

Entities, Attributes, and Entity Sets

  • Entity: An object that exists and is distinguishable from other objects.
  • Attribute: Property that describes an entity.
  • Entity Set: A set of entities of the same type that share the same attributes.

Relationships and Relationship Sets

  • Relationship: An association among several entities.
  • Relationship Set: A set of relationships of the same type.

Extended ER Features

  • Generalization: Abstracting common features.
  • Specialization: Creating subgroups from an entity set.
  • Aggregation: Treating a relationship set as an entity set.

Conceptual Design with the ER Model

Involves mapping user requirements to an ER model, considering constraints and requirements before moving to a relational schema.

Logical Database Design

Converting ER diagrams into relational tables. This phase focuses on logical data structures without considering physical storage.
