Routing and Switching

Introduction

Routing and Switching are fundamental concepts in computer networking. They determine how data moves through a network, from source to destination, efficiently and reliably.

Switching

Switching refers to the process of directing data packets between devices within the same network. It operates mainly at Layer 2 (Data Link Layer) of the OSI model.

Types of Switching

  • Circuit Switching: Establishes a dedicated communication path between two devices (used in traditional telephone networks).
  • Packet Switching: Data is divided into packets and each packet can take any path to the destination (used in the Internet).
  • Message Switching: Whole messages are switched and stored until the next link is available.

Network Switch

A network switch connects devices in a LAN and uses MAC addresses to forward data to the correct destination. Unlike hubs, switches are intelligent devices that reduce collisions and improve network efficiency.

Routing

Routing is the process of selecting a path for traffic in a network, or across multiple networks. It typically operates at Layer 3 (Network Layer) of the OSI model.

Types of Routing

  • Static Routing: Routes are manually configured and do not change unless manually updated.
  • Dynamic Routing: Routers automatically adjust their paths based on network conditions using protocols.

Routing Algorithms

  • Distance Vector Routing: Routers inform their neighbors of topology changes periodically. Example: RIP (Routing Information Protocol).
  • Link State Routing: Routers have a full map of the network and compute the shortest path. Example: OSPF (Open Shortest Path First).

Routing Protocols

  • RIP (Routing Information Protocol): A simple Distance Vector protocol with a maximum hop count of 15.
  • OSPF (Open Shortest Path First): A Link State protocol using Dijkstra's algorithm for the shortest path.
  • BGP (Border Gateway Protocol): The protocol that makes the internet work by exchanging routing information between ISPs.
  • EIGRP (Enhanced Interior Gateway Routing Protocol): A Cisco proprietary protocol combining features of both Distance Vector and Link State routing.

Differences between Routing and Switching

Aspect Switching Routing
Function Forwards data within the same network (LAN) Forwards data between different networks
Layer Data Link Layer (Layer 2) Network Layer (Layer 3)

Addressing MAC Address IP Address

Conclusion

Routing and Switching are essential for building efficient and scalable networks. While switching helps devices within a network communicate, routing enables communication between different networks, including the internet.
