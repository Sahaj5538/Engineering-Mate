UNIT II: Basic Computer Organization and Microprogrammed Control

Basic Computer Organization and Design

Instruction Codes

Instruction codes represent the operation part of machine language instructions, allowing computers to understand and execute commands.

Computer Registers

  • Accumulator (AC): Temporary data storage for calculations.
  • Program Counter (PC): Holds address of next instruction.
  • Instruction Register (IR): Holds the instruction currently being executed.
  • Memory Address Register (MAR): Holds address of memory location.
  • Memory Buffer Register (MBR): Holds data from or to memory.

Computer Instructions

Instructions can be classified as memory-reference, register-reference, or input-output instructions based on their operation.

Timing and Control

Responsible for synchronizing all internal operations in the CPU based on clock cycles using control signals.

Instruction Cycle

  • Fetch
  • Decode
  • Execute
  • Store/Write-back

Memory Reference Instructions

Instructions that involve operations where data is fetched or stored to memory, such as LDA (Load Accumulator), STA (Store Accumulator), ADD, SUB, etc.

Input-Output and Interrupt

Input-Output operations transfer data between CPU and peripherals. Interrupts are signals that temporarily halt CPU operations to service an urgent task.

Complete Computer Description

A description that includes data flow, control flow, register setup, memory usage, and instruction cycles to implement a simple basic computer system.

Microprogrammed Control

Control Memory

Control memory holds control information (microinstructions) necessary to generate control signals.

Address Sequencing

The process of determining the next microinstruction address based on the conditions and control signals.

Microprogram Example

A sample sequence of microinstructions that implement a macroinstruction (such as ADD or LOAD).

Design of Control Unit

Design involves developing control memory, microinstruction formats, and methods for sequencing through microinstructions.

Microprogram Sequencer

A unit that decides the address of the next microinstruction, based on the current instruction and condition flags.

Hardwired Control vs Microprogrammed Control

  • Hardwired Control: Control signals are generated by combinational logic circuits. Faster but less flexible.
  • Microprogrammed Control: Control signals are stored in memory and accessed like data, making it more flexible but slower.
