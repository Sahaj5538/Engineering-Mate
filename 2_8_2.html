XML (eXtensible Markup Language)

1. Introduction to XML

XML stands for eXtensible Markup Language. It is designed to store and transport data. XML focuses on the structure of the data rather than how it looks. It is both human-readable and machine-readable.

2. Defining XML Tags, Attributes, and Values

In XML, you define your own tags. Tags must be properly nested and closed.

<book id="101">
    <title>XML Fundamentals</title>
    <author>John Doe</author>
</book>
  • Tags: e.g., <title>, <author>
  • Attributes: e.g., id="101" inside the <book> tag
  • Values: The data between tags like XML Fundamentals

3. Document Type Definition (DTD)

DTD defines the structure and the legal elements and attributes of an XML document.

<!DOCTYPE book [
    <!ELEMENT book (title, author)>
    <!ELEMENT title (#PCDATA)>
    <!ELEMENT author (#PCDATA)>
]>

4. XML Schemas

XML Schema is an advanced way to define the structure of XML documents. It is more powerful than DTD.

<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema">
  <xs:element name="book">
    <xs:complexType>
      <xs:sequence>
        <xs:element name="title" type="xs:string"/>
        <xs:element name="author" type="xs:string"/>
      </xs:sequence>
    </xs:complexType>
  </xs:element>
</xs:schema>

5. Document Object Model (DOM)

DOM represents the structure of XML documents as a tree of objects. Each element, attribute, and text is represented as a node in this tree.

6. XHTML

XHTML is a stricter version of HTML that follows XML syntax rules, ensuring well-formedness.

7. Parsing XML Data: DOM and SAX Parsers

  • DOM Parser: Loads the entire XML document into memory and allows easy traversal and modification.
  • SAX Parser: Event-driven, reads XML sequentially, and is faster and memory-efficient.

DOM Parsing Example (JavaScript):

let parser = new DOMParser();
let xmlDoc = parser.parseFromString(xmlString, "text/xml");
console.log(xmlDoc.getElementsByTagName("title")[0].childNodes[0].nodeValue);

AngularJS

1. Introduction to AngularJS

AngularJS is a JavaScript-based open-source front-end web framework mainly maintained by Google. It simplifies the development of dynamic web applications by extending HTML's capabilities with new attributes and expressions.

2. Expressions

AngularJS expressions are like JavaScript expressions and can be written inside double curly braces {{ }}.

<div ng-app="">
  <p>{{ 5 + 5 }}</p>
</div>

3. Modules

Modules define the application and its dependencies.

var app = angular.module('myApp', []);

4. Directives

Directives are special attributes like ng-model, ng-bind, ng-app that extend HTML capabilities.

<div ng-app="" ng-init="firstName='John'">
  <p>Name: <input type="text" ng-model="firstName"></p>
  <p>Hello {{ firstName }}!</p>
</div>

5. AngularJS HTML DOM

AngularJS binds data to HTML using the scope object. It can dynamically update the DOM elements without reloading the page.

6. Events

AngularJS provides event directives to handle user actions such as clicks, mouse movements, etc.

<button ng-click="count = count + 1">Click Me!</button>

7. Forms

AngularJS provides validation support for form controls:

  • ng-model - Binds input fields to model data
  • ng-required - Makes an input field required
  • ng-minlength and ng-maxlength - Restricts input length
<form name="myForm">
  Name: <input type="text" name="userName" ng-model="userName" required>
  <span ng-show="myForm.userName.$touched && myForm.userName.$invalid">Name is required.</span>
</form>
