UNIT III: Central Processing Unit Organization and Computer Arithmetic

Central Processing Unit Organization

General Register Organization

In general register organization, all CPU operations are performed using multiple processor registers. It increases speed by avoiding frequent memory access.

  • ALU connected to a set of registers and buses.
  • Registers include accumulator, data register, address register, etc.
  • Instructions specify register operands.
Example: ADD R1, R2 → R1 = R1 + R2 (Adds contents of R2 to R1)

Stack Organization

Stack is a last-in-first-out (LIFO) memory structure used for expression evaluation and function call management.

  • Stack Pointer (SP) points to the top of the stack.
  • Operations: PUSH (insert), POP (remove).
Example: PUSH X → SP = SP - 1; Stack[SP] = X (Pushes value X onto the stack.)

Instruction Formats

Instruction format specifies the layout of bits that form an instruction. Typical fields:

  • Opcode: Operation to be performed
  • Operand: Source/target data or register address
  • Address: Memory address if required

Addressing Modes

Defines how the effective address of the operand is calculated. Types:

  • Immediate: Operand is part of instruction.
  • Direct: Address field contains the effective address.
  • Indirect: Address field points to the address holding the effective address.
  • Indexed: Address = Base + Index Register.

Data Transfer and Manipulation

Data transfer instructions move data between memory, registers, and I/O devices. Manipulation instructions perform operations like AND, OR, SHIFT, ADD, etc.

Program Control

Program control instructions alter the flow of execution, including branch, jump, call, and return operations.

Example: JMP 300 (Changes PC to 300.)

Reduced Instruction Set Computer (RISC)

RISC architecture simplifies instruction sets to achieve higher performance. Characteristics:

  • Simple instructions
  • Fixed instruction size
  • Load/store architecture
  • Fewer addressing modes
Example: RISC example CPU: ARM, MIPS

Computer Arithmetic

Addition and Subtraction

Binary addition and subtraction use full-adders and subtractors.

  • Binary Addition: Same as decimal but carries when sum ≥ 2.
  • Binary Subtraction: Uses complement methods (1's or 2's complement).
Example: 1011 + 1101 = 11000

Multiplication Algorithms

Binary multiplication is done by successive addition and shifting. Methods:

  • Booth’s Multiplication Algorithm
  • Bit-pair recoding
Example: Multiply 101 by 11 → Shift and add partial products.

Floating-Point Arithmetic Operations

Floating-point numbers are represented using mantissa and exponent. IEEE 754 format is commonly used.

  • Normalization required after operations.
  • Handling rounding and overflow carefully.

BCD Adder

BCD (Binary Coded Decimal) Adder is used to add two BCD numbers.

  • If sum > 9, add 6 to correct.
  • Required to adjust binary results to valid BCD codes.
Example: 9 (1001) + 8 (1000) = 17 (0001 0111) → Add 6 (0110) to get 0010 0011 (23 in BCD)
