Linked Lists

Introduction to Lists

Dynamic Memory Allocation

Dynamic Memory Allocation allows the program to allocate memory during runtime as needed, providing flexibility in handling varying data sizes. Linked Lists use dynamic memory to create nodes as required, optimizing memory use.

Self-referential Structures

A self-referential structure is a structure that contains a pointer to another structure of the same type. In linked lists, each node typically contains data and a pointer to the next node.

Single Linked List (SLL)

Advantages and Disadvantages of SLL

  • Advantages:
    • Dynamic size
    • Efficient insertions/deletions
    • Better memory utilization
  • Disadvantages:
    • Extra memory for pointer storage
    • Sequential access only (no random access)
    • More complex operations compared to arrays

SLL vs Arrays

  • Arrays require contiguous memory, whereas Linked Lists can use scattered memory blocks.
  • Arrays allow random access using indices; Linked Lists allow sequential access only.
  • Insertion/deletion is expensive in arrays but efficient in linked lists.

Memory Representation

In a Single Linked List, each node contains:

  • Data field: stores the data
  • Next pointer: points to the next node in the list
The last node points to NULL, indicating the end of the list.

Operations on Linked Lists

Insertion

Inserting a new node can be done at:

  • The beginning of the list
  • At a specific position
  • At the end of the list
The pointers are adjusted to accommodate the new node.

Deletion

Deletion involves removing a node from:

  • The beginning
  • A specified position
  • The end
Care must be taken to properly update the links and free memory.

Display

To display the list, we traverse from the head node, following the next pointers, and print each node’s data until reaching NULL.

Search

Searching in a linked list involves traversing node by node and comparing the target value with each node’s data field.

Applications

Implementation of Stack using Linked List

A stack can be implemented using a linked list where:

  • push operation adds a new node at the beginning (top of the stack)
  • pop operation removes the node from the beginning

Implementation of Queue using Linked List

A queue can be implemented using a linked list where:

  • enqueue operation adds a new node at the rear
  • dequeue operation removes a node from the front

Other Types of Linked Lists

Circular Linked List

In a Circular Linked List, the last node points back to the first node, making the list circular. It can be singly or doubly linked. It is useful for applications that require a cyclic traversal.

Doubly Linked List

A Doubly Linked List has two pointers in each node:

  • One pointing to the next node
  • One pointing to the previous node
It allows traversal in both forward and backward directions but uses extra memory for the additional pointer.
