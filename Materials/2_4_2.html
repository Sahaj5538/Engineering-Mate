SQL Queries and Constraints

SQL Data Definition

SQL provides commands to define database schema including creating, altering, and dropping tables and other database objects.

Types of SQL Commands

  • DDL (Data Definition Language): CREATE, ALTER, DROP
  • DML (Data Manipulation Language): INSERT, UPDATE, DELETE, SELECT
  • DCL (Data Control Language): GRANT, REVOKE
  • TCL (Transaction Control Language): COMMIT, ROLLBACK, SAVEPOINT

Form of Basic SQL Query

Typical form:

SELECT attribute_list
FROM relation_list
WHERE condition;

SQL Operators

  • Arithmetic Operators: +, -, *, /
  • Comparison Operators: =, <>, <, >, <=, >=
  • Logical Operators: AND, OR, NOT

Set Operators

  • UNION: Combines results from two queries (removes duplicates).
  • UNION ALL: Combines results (includes duplicates).
  • INTERSECT: Retrieves common records.
  • MINUS/EXCEPT: Returns records from the first query that are not present in the second.

Nested Queries

Queries written inside another query, often used with IN, EXISTS, ALL, ANY keywords.

Aggregate Operators

  • SUM() - Calculates the total.
  • AVG() - Calculates the average.
  • MAX() - Finds the maximum value.
  • MIN() - Finds the minimum value.
  • COUNT() - Counts the number of rows.

NULL Values

SQL uses NULL to represent missing or unknown data. Functions like IS NULL or IS NOT NULL are used to check for NULLs.

Integrity Constraints Over Relations

  • Primary Key: Uniquely identifies a record.
  • Foreign Key: Ensures referential integrity between tables.
  • NOT NULL: Ensures that a column cannot have a NULL value.
  • CHECK: Ensures that all values in a column satisfy a specific condition.
  • UNIQUE: Ensures all values in a column are different.

Joins

  • INNER JOIN: Returns records with matching values in both tables.
  • LEFT JOIN: Returns all records from the left table and matched records from the right table.
  • RIGHT JOIN: Returns all records from the right table and matched records from the left table.
  • FULL JOIN: Returns records when there is a match in either table.

Introduction to Views

A View is a virtual table based on the result set of a SQL query.

CREATE VIEW view_name AS
SELECT column1, column2, ...
FROM table_name
WHERE condition;

Destroying / Altering Tables and Views

  • DROP TABLE/View: Removes the table/view completely.
  • ALTER TABLE: Modifies the structure of an existing table.

Cursors

A Cursor is a database object used to retrieve, manipulate, and navigate through a result set row by row.

Triggers and Active Databases

A Trigger is a SQL procedure that initiates an action in response to a database event (INSERT, UPDATE, DELETE).

CREATE TRIGGER trigger_name
AFTER INSERT ON table_name
FOR EACH ROW
BEGIN
    -- trigger logic
END;
