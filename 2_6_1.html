UNIT I: Operating System Overview and Process Management

Operating System Overview

Objectives and Functions

An Operating System (OS) acts as an intermediary between the user and computer hardware. Main objectives:

  • Efficient management of resources (CPU, memory, I/O devices)
  • Providing a convenient user interface
  • Ensuring security and data integrity
  • Facilitating communication between applications and hardware

Computer System Architecture

Architecture includes the CPU, memory, I/O devices, and storage systems. Types of architectures:

  • Single-Processor Systems
  • Multiprocessor Systems (Parallel systems)
  • Clustered Systems

Evolution of Operating Systems

  • Early systems (Batch systems)
  • Multiprogramming Systems
  • Time-Sharing Systems
  • Distributed Systems
  • Modern Mobile and Cloud OS

System Services

OS provides various services such as:

  • Program execution
  • I/O operations
  • File system manipulation
  • Communication services
  • Error detection and security

System Calls

System calls are the programming interface to the services provided by the OS. Example categories:

  • Process control (e.g., fork, exec)
  • File management (e.g., open, close, read, write)
  • Device management
  • Information maintenance
  • Communication
Example: fork() - creates a new process in UNIX/Linux systems.

System Programs

System programs provide a convenient environment for program development and execution. Examples include compilers, assemblers, file manipulation programs, and system utilities.

OS Structure

Common OS Structures:

  • Simple Structure (e.g., MS-DOS)
  • Layered Approach
  • Microkernel Systems
  • Modular Approach

Virtual Machines

A virtual machine provides an abstraction of the computer hardware to multiple environments. Examples:

  • VMware
  • VirtualBox
  • Hyper-V

Process Management

Process Concepts

A process is a program in execution, consisting of program code, data, and a set of resources. Key components:

  • Process Control Block (PCB)
  • States: New, Ready, Running, Waiting, Terminated

CPU Scheduling

CPU scheduling selects which process will be executed next on the CPU. Criteria include:

  • CPU utilization
  • Throughput
  • Turnaround time
  • Waiting time
  • Response time

Scheduling Algorithms

  • FCFS (First-Come, First-Served)
  • SJF (Shortest Job First)
  • Round Robin
  • Priority Scheduling
  • Multilevel Queue Scheduling
Example: Round Robin Scheduling: Each process gets a small unit of CPU time (time quantum), and then moved to the back of the queue.

Preemptive vs Non-Preemptive Scheduling

  • Preemptive: A running process can be interrupted.
  • Non-Preemptive: A running process cannot be interrupted until it finishes.

Threads

Threads are the smallest sequence of programmed instructions that can be managed independently by a scheduler.

  • Multithreading improves concurrency.
  • Multiple threads can exist within one process.

Multithreading Models

  • Many-to-One Model: Many user-level threads map to one kernel thread.
  • One-to-One Model: Each user-level thread maps to one kernel thread.
  • Many-to-Many Model: Many user threads map to many kernel threads.
