Relational Model and Storage & Indexing

Relational Model

Introduction to Relational Model

The relational model represents the database as a collection of relations (tables). Each relation consists of tuples (rows) and attributes (columns).

Basic Structure

  • Relation: A table with columns and rows.
  • Tuple: A single row in a relation.
  • Attribute: A named column of a relation.
  • Domain: A set of allowed values for an attribute.

Database Schema

A schema is the logical structure of the database, including the definition of tables, columns, types, and relationships.

Keys

  • Primary Key: A field (or fields) that uniquely identifies each tuple in a table.
  • Candidate Key: A set of fields that could serve as a primary key.
  • Foreign Key: A field in one table that links to the primary key in another table.
  • Super Key: A set of one or more attributes that uniquely identify a tuple.

Relational Algebra

Relational algebra is a procedural query language that works on the relational model. Main operations include:

  • Selection (σ): Selects tuples that satisfy a given predicate.
  • Projection (π): Selects certain columns from a relation.
  • Union (⋃): Combines the tuples of two relations.
  • Set Difference (-): Finds tuples in one relation but not in another.
  • Cartesian Product (×): Combines every tuple of one relation with every tuple of another relation.
  • Rename (ρ): Renames the relation or its attributes.
  • Join (⨝): Combines related tuples from two relations.

Relational Calculus

Relational calculus is a non-procedural query language where users specify what they want, not how to compute it.

  • Tuple Relational Calculus (TRC)
  • Domain Relational Calculus (DRC)

Storage and Indexing

File Organizations and Indexing

File organization refers to the way data is stored in files and how records are organized on storage devices.

Overview of Indexes

Indexes improve the speed of data retrieval operations on a database table at the cost of additional storage space and maintenance time.

Types of Indexes

  • Primary Index: An index on a set of fields that includes the primary key.
  • Secondary Index: An index on non-primary key fields.
  • Clustered Index: Alters the way records are stored in the database (sorted according to the index).
  • Non-clustered Index: Creates a separate structure from the data table.

Index Data Structures

  • B+ Trees: Balanced tree data structure that maintains sorted data and allows searches, sequential access, insertions, and deletions in logarithmic time.
  • Hash Indexing: Uses a hash table to locate a data record given its search key.

Tree-Structured Indexing

Tree-structured indexes, like B-trees and B+ trees, are used to maintain sorted order of data and allow fast searches, insertions, deletions, and sequential access.

Hash-Based Indexing

Hash-based indexing uses a hash function to compute the address of a page where a record might be stored.
