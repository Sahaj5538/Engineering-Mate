Gate-Level Minimization

Karnaugh Maps (2 to 5 Variables)

A Karnaugh Map (K-map) is a graphical tool used to simplify Boolean expressions and minimize the number of logic gates needed to implement a circuit.

Features:

  • Represents truth tables in a two-dimensional grid format.
  • Adjacent cells differ by only one bit (Gray code sequence).
  • Grouping adjacent 1s (or 0s for POS) reduces expressions.

K-map Sizes:

  • 2-variable K-map: 4 cells (2x2 grid)
  • 3-variable K-map: 8 cells (2x4 grid)
  • 4-variable K-map: 16 cells (4x4 grid)
  • 5-variable K-map: 32 cells (split into two 4-variable K-maps)

Groups must be powers of two: 1, 2, 4, 8, etc., and can wrap around the edges of the map.

Don’t Care Conditions

Don’t Care Conditions are scenarios where the output value (0 or 1) does not affect the functionality of the circuit.

Usage:

  • Represented by "X" in the K-map.
  • Can be treated as 0 or 1 to form larger groups, allowing greater simplification of Boolean expressions.
  • Typically occur in incomplete truth tables, unused input combinations, or redundant system states.

NAND/NOR Implementations

Any Boolean function can be implemented using only NAND or only NOR gates because these gates are "functionally complete."

NAND Implementation:

  • Apply the concept of double negation (A = (A'') ).
  • Convert the given expression into NAND logic by appropriate inversions.

NOR Implementation:

  • Similar to NAND implementation but using NOR gates.
  • Use DeMorgan’s Theorems to rewrite AND/OR operations as NOR operations.

Using only NAND/NOR gates is beneficial for cost, design simplicity, and uniformity in IC (Integrated Circuit) design.

Exclusive-OR Functions

The Exclusive-OR (XOR) function is a Boolean function that outputs true (1) when the inputs are different.

XOR Gate Symbol: ⊕

Properties:

  • A ⊕ 0 = A
  • A ⊕ 1 = A'
  • A ⊕ A = 0
  • A ⊕ A' = 1

Expression:

For two variables A and B, the XOR function is expressed as:

A ⊕ B = A'B + AB'

Applications:

  • Used in arithmetic operations like addition (half adder, full adder).
  • Used for error detection and correction.
  • Useful in parity generation and checking.
