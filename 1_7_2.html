Stacks and Queues

Stacks

Introduction to Data Structures and Their Types

Data Structures are methods of organizing and storing data so that they can be accessed and modified efficiently. Common types include Arrays, Linked Lists, Stacks, Queues, Trees, and Graphs.

Stack is a linear data structure that follows the Last In First Out (LIFO) principle, meaning the last element added is the first one to be removed.

Stack Operations

  • push: Add an element to the top of the stack.
  • pop: Remove the top element from the stack.
  • peek: View the top element without removing it.
  • display: Show all the elements in the stack.

Representation and Implementation: Using Arrays

Stacks can be implemented using arrays. We maintain a variable (typically called top) to keep track of the index of the last inserted element. Operations are performed relative to top.

Stack Applications

  • Recursion: Function calls use the stack internally to store function call details like parameters and local variables.
  • Infix to Postfix Transformation: Converting expressions from infix notation (e.g., A+B) to postfix notation (e.g., AB+), using stack to manage operators and parentheses.
  • Evaluating Postfix Expressions: Postfix expressions are evaluated using a stack by pushing operands and applying operators as they appear.

Queues

Queue Operations

  • enqueue: Add an element to the rear (end) of the queue.
  • dequeue: Remove an element from the front of the queue.
  • display: Show all the elements in the queue from front to rear.

Representation and Implementation: Using Arrays

Queues can be implemented using arrays. Two variables, front and rear, are used to keep track of the start and end positions of the queue respectively. Elements are added at the rear and removed from the front.

Applications of Queues

  • Managing requests on a single shared resource (e.g., printer scheduling).
  • Handling data packets in networking (e.g., router queues).
  • Implementing First-Come-First-Served (FCFS) scheduling policies.

Circular Queues

In a Circular Queue, the last position is connected back to the first position to make a circle. It solves the problem of unused space in the linear queue (array-based implementation).

Representation and Implementation

Using arrays, after the last index, the rear pointer wraps around to index 0 if there is empty space at the beginning of the array. The condition for a full queue is when the next position of rear is front.
